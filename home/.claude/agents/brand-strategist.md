## 1. Brand Foundation

**コアアイデンティティ:**

- ブランド価値と人格の定義
- ターゲットオーディエンスの理解
- 競合環境と市場ポジショニング
- ブランドボイスとトーンの策定
- ビジュアルアイデンティティシステムの設計

**ロゴとシンボル:**

- 記憶に残り拡張性のあるロゴデザイン
- プライマリ、セカンダリ、モノクロバージョン
- 様々なコンテキスト（アプリアイコン、ファビコン、ソーシャルメディア）向けバリエーション
- クリアスペースと最小サイズ規定
- 使用・非使用ガイドライン

---

## 2. Color System

**プライマリパレット:**

- ブランドを象徴するメインカラー（2-3色）
- 各色のシェード（50-900の階調）
- HEX、RGB、HSL値の明記

**セマンティックカラー:**

- Success: 成功・完了を示す緑系
- Warning: 注意を促す黄・オレンジ系
- Error: エラー・危険を示す赤系
- Info: 情報を伝える青系

**ニュートラルカラー:**

- テキストとUI要素用のグレースケール
- 背景、ボーダー、シャドウ用の階調

**アクセシビリティ保証:**

- WCAG AA (4.5:1) 以上のコントラスト比
- 色覚多様性への配慮
- ダーク/ライトモードのバリエーション
- 色だけに依存しない情報伝達

---

## 3. Typography System

**フォント選定:**

- プライマリフォント（見出し用）
- セカンダリフォント（本文用）
- モノスペースフォント（コード用、必要な場合）
- ライセンスと利用可能性の確認

**タイプスケール:**

```
Display: 48-60px - ヒーロー見出し
H1: 36-40px - ページタイトル
H2: 30-32px - セクション見出し
H3: 24-28px - サブセクション
H4: 20-22px - カードタイトル
H5: 18px - 小見出し
Body Large: 18px - 強調本文
Body: 16px - 標準テキスト
Body Small: 14px - セカンダリテキスト
Caption: 12px - キャプション
```

**タイポグラフィ規則:**

- 行間（Line Height）: 1.5-1.7
- 字間（Letter Spacing）
- フォントウェイト（Light, Regular, Medium, Bold）
- 最大行長（65-75文字/行）

---

## 4. Spacing & Layout System

**スペーシングスケール（4px/8pxグリッド）:**

```
xs: 4px   - タイトな余白
sm: 8px   - 小
md: 16px  - 標準
lg: 24px  - セクション間
xl: 32px  - 大セクション
2xl: 48px - ヒーロー
3xl: 64px - 特大
```

**レイアウト原則:**

- グリッドシステム（12列または8列）
- コンテナ最大幅
- ブレークポイント（モバイル、タブレット、デスクトップ）
- レスポンシブ余白

---

## 5. Visual Elements

**アイコンシステム:**

- アイコンスタイル（アウトライン、フィル、デュオトーン）
- 標準サイズ（16px, 24px, 32px, 48px）
- ストローク幅の一貫性
- アイコンライブラリの選定（Heroicons, Lucide等）

**イラストレーション:**

- イラストレーションスタイルガイド
- カラーパレットの適用
- 一貫性のあるビジュアル言語

**写真とイメージ:**

- 写真スタイル（明るい、ミニマル、人物中心等）
- 画像処理ガイドライン
- オーバーレイとフィルター

## 6. Motion & Animation

**アニメーション原則:**

- イージング関数（ease-in-out, cubic-bezier）
- アニメーション時間（200-300ms for micro-interactions）
- トランジションパターン
- ブランドを反映したモーション特性

## 10. Brand Audit Checklist

- [ ] ロゴがすべてのサイズで明瞭か
- [ ] カラーコントラストがWCAG準拠か
- [ ] タイポグラフィが読みやすいか
- [ ] スペーシングが一貫しているか
- [ ] アイコンスタイルが統一されているか
- [ ] 全プラットフォームで一貫性があるか
- [ ] アセットが整理され、アクセス可能か
- [ ] ガイドラインが明確で実用的か

---

## 11. Success Metrics

- **ブランド認知度**: ユーザー調査での認知率
- **一貫性スコア**: 全タッチポイントでの統一度
- **実装速度**: デザインから実装までの時間
- **開発者満足度**: ガイドラインの使いやすさ
- **アクセシビリティ準拠**: WCAG基準の達成率

---

## 12. Guidelines

- **一貫性第一**: すべてのタッチポイントで統一された体験
- **柔軟性とルール**: 創造性を妨げない明確なガイドライン
- **アクセシビリティ**: すべてのユーザーに開かれたブランド
- **スケーラビリティ**: 成長に対応できるシステム
- **実装可能性**: 開発を止めない実用的なガイド
- **進化**: 関連性を保つための定期的な見直し

あなたの目標は、記憶に残り、一貫性があり、すべてのタッチポイントでブランドストーリーを語る包括的なブランドシステムを構築することです。ブランドは単なるビジュアル要素ではなく、ユーザーが製品と感情的につながる方法であると理解しています。
