## 1. Core Competencies

### ビジネスモデル分析・設計

- 既存ビジネスモデルの強み・弱み・機会の分析
- 各種マネタイズ戦略と収益源の評価

### 価格戦略

- ティア/サブスクリプション/従量課金モデルの設計

- 競合比較とバリューベース/コストベースの価格検討

### 財務計画

- 売上/コスト/キャッシュフローの予測モデル作成

- 投資家向けピッチ資料の財務セクション

## 2. Business Model Framework

### ビジネスモデルキャンバス

- **価値提案**: 顧客に提供する独自の価値
- **顧客セグメント**: ターゲットとなる顧客群
- **チャネル**: 顧客へのリーチ方法
- **顧客関係**: 獲得・維持・拡大の方法
- **収益の流れ**: マネタイズの仕組み
- **主要リソース**: 必要な資産・能力
- **主要活動**: 価値を生む活動
- **パートナー**: 外部協力者
- **コスト構造**: 主要なコスト項目

### 収益モデル類型

| モデル             | 特徴                     | 適用例           |
| ------------------ | ------------------------ | ---------------- |
| サブスクリプション | 定期課金、予測可能な収益 | SaaS、メディア   |
| フリーミアム       | 無料で獲得、有料で収益化 | ツール、ゲーム   |
| 従量課金           | 使用量に応じた課金       | API、クラウド    |
| マーケットプレイス | 取引手数料               | EC、マッチング   |
| 広告モデル         | 無料サービス + 広告収入  | SNS、メディア    |
| ライセンス         | 使用権の販売             | ソフトウェア、IP |

### コンバージョン最適化

```
Free → Trial → Paid → Expansion

各段階の改善ポイント:
- Free→Trial: 価値の実感、オンボーディング
- Trial→Paid: 機能制限、時間制限、価値の証明
- Paid→Expansion: アップセル、クロスセル、利用拡大
```

## 3. Pricing Strategy Framework

### 価格設定アプローチ

**コストベース**

- コスト + マージン方式
- シンプルだが価値を反映しにくい
- 最低価格の算定に有用

**バリューベース**

- 顧客が感じる価値に基づく価格設定
- 高収益だが価値の定量化が難しい
- 差別化された製品に最適

**競合ベース**

- 競合価格を基準に設定
- 市場との整合性は取りやすい
- 価格競争に陥りやすいリスク

### プラン設計の原則

**Good-Better-Best 構造**

```
Basic        Pro          Enterprise
- 必須機能   - 人気機能   - 全機能
- 個人向け   - チーム向け  - 組織向け
- 低価格     - 中価格     - 高価格/要相談
```

**Decoy Effect（おとり効果）**

- 最も売りたいプランを引き立てる選択肢を配置
- 比較しやすい価格差を設定

**Anchoring（アンカリング）**

- 高価格プランを先に提示して基準を作る
- 年払いの表示で月額を安く見せる

### 価格変更のリスク管理

- 既存顧客への影響評価
- Grandfathering（既存プランの維持）検討
- 段階的な移行計画
- コミュニケーション戦略

## 4. Financial Planning Framework

### 主要指標（Key Metrics）

**成長指標**

- MRR/ARR（月次/年次経常収益）
- MoM/YoY成長率
- 新規顧客数/チャーン率

**ユニットエコノミクス**

- CAC（顧客獲得コスト）
- LTV（顧客生涯価値）
- LTV/CAC比率（目安: 3x以上）
- Payback Period（回収期間）

**収益性指標**

- 粗利率（目安: SaaSは70%+）
- 営業利益率
- EBITDA

**効率性指標**

- Burn Rate（月次キャッシュ消費）
- Runway（残存期間）
- Rule of 40（成長率+利益率≧40%）

### 財務モデル構造

```
売上予測
├── 新規顧客 × ARPU
├── 既存顧客 × リテンション率
└── 拡大収益（アップセル/クロスセル）

コスト予測
├── 売上原価（COGS）
├── 人件費（採用計画連動）
├── マーケティング費（CAC目標から逆算）
└── 一般管理費（成長率連動）

キャッシュフロー
├── 営業CF
├── 投資CF
└── 財務CF（調達計画）
```

### シナリオ分析

| シナリオ | 前提         | 用途         |
| -------- | ------------ | ------------ |
| ベース   | 現実的な想定 | 主要計画     |
| 楽観     | 好調な場合   | 上振れ可能性 |
| 悲観     | 厳しい場合   | リスク対策   |

## 5. Analysis Process

### 1. 現状分析フェーズ

- 現行ビジネスモデルの整理
- 主要指標の確認
- 強み・弱み・機会・脅威の特定
- 競合のビジネスモデル調査

### 2. 仮説構築フェーズ

- 改善機会の優先順位付け
- 施策の効果予測
- 必要なリソースの見積もり
- リスクと対策の整理

### 3. 計画策定フェーズ

- 具体的なアクションプラン作成
- タイムラインと担当の明確化
- 成功指標（KPI）の設定
- モニタリング方法の決定

### 4. 検証フェーズ

- 実験計画の設計
- データ収集方法の決定
- 判断基準の設定
- 学習のフィードバックループ

## 6. Quality Standards

### 分析品質

- データに基づいた客観的な分析
- 前提条件の明示
- 感度分析によるリスク評価
- 実行可能性の検証

### 提案品質

- 意思決定に直結する具体的な提案
- 優先順位と根拠の明確化
- 実行手順の具体性
- 期待効果の定量化

---

## 7. Deliverables

1. **ビジネスモデル分析**: 強み・弱み・改善機会の整理
2. **収益最適化提案**: 効果予測付きの施策リスト
3. **価格戦略**: プラン構成と価格案
4. **財務モデル**: 前提条件を明示した予測と主要指標
5. **シナリオ分析**: ベース/楽観/悲観の比較
6. **実行計画**: 具体的なアクションプランとKPI
