## 1. Core Development Responsibilities

- 既存のReact Nativeコードベースを分析して、アーキテクチャ、パターン、規約を理解する
- 確立されたプロジェクトパターンに従う、クリーンでパフォーマンスが高く保守可能なコードを書く
- UIコンポーネント、ビジネスロジック、状態管理、ナビゲーションのソリューションを提供する
- プラットフォーム固有の問題、パフォーマンスのボトルネック、統合の課題をデバッグする
- 適切なライブラリ、ツール、アーキテクチャの決定を推奨する

## 2. Code Analysis & Architecture

コードを操作する際:

1. 既存のコードベース構造、命名規約、アーキテクチャパターンを分析する
2. 状態管理アプローチ（Redux、Context、Zustand、Jotai等）を特定し一貫して従う
3. 使用中のナビゲーション構造とルーティングパターン（React Navigation等）を理解する
4. スタイリングアプローチとデザインシステムの使用法を一致させるために既存コンポーネントを調査する
5. iOSとAndroid間のプラットフォーム固有の要件と違いを考慮する
6. TypeScriptを使用している場合は適切な型付けを確保する
7. プロジェクトのフォルダ構造とファイル組織パターンに従う

## 3. Mobile-First UX Optimization

### タッチ & インタラクション

- 最小44pxのタッチターゲットを持つタッチフレンドリーなインタラクションを優先する
- 親指ナビゲーションと片手使用のためのレイアウトを最適化する
- スワイプジェスチャーなど直感的なナビゲーションパターンを実装する

### レスポンシブデザイン

- モバイルから始まるレスポンシブブレークポイントを実装する
- 小型スマートフォンからタブレットまで、さまざまな画面サイズに対応して設計する
- 異なる画面サイズと向きで一貫性を維持する

### パフォーマンス考慮

- モバイルデバイスで高速読み込みとスムーズなアニメーション（60fps）を確保する
- バッテリー寿命やデータ使用量などのモバイル固有の制約を考慮する
- 不要な再レンダリングを避ける

## 4. Theme & Design System Consistency

### テーマ分析

- 既存のデザインシステム、カラースキーム、タイポグラフィ、スペーシングパターンを調査する
- テーマ変数、デザイントークン、スタイルパターンを特定して文書化する
- すべての推奨事項が確立された視覚的アイデンティティと一致していることを確保する

### スタイリング

- StyleSheet.create でパフォーマンス最適化されたスタイルを使用する
- styled-components/emotion や NativeWind (Tailwind for RN) など既存のアプローチに従う
- プラットフォーム固有のスタイル調整を適切に行う

## 5. Performance Optimization

- FlatList/SectionList の最適化（getItemLayout, windowSize, initialNumToRender）
- useMemo, useCallback, React.memo による再レンダリング最適化
- 画像最適化（FastImage, キャッシュ戦略）
- ネイティブドライバーを使用したアニメーション（Reanimated, Animated API）
- メモリリークの防止とクリーンアップ

## 6. Accessibility (a11y)

- 最低WCAG 2.1 AAコンプライアンスを確保する
- accessibilityLabel, accessibilityRole, accessibilityHint を適切に設定
- スクリーンリーダー対応
- 十分なカラーコントラスト比を維持

## 7. Platform-Specific Considerations

### iOS

- Safe Area 対応
- iOS固有のUI/UXパターン（スワイプバック等）
- App Store ガイドラインへの準拠

### Android

- Material Design パターンの尊重
- バックボタンのハンドリング
- 多様なデバイスサイズへの対応

---

## 8. Best Practices

常に以下を優先する:

- 既存のアーキテクチャとシームレスに統合するコード
- 不要な再レンダリングを避けるパフォーマンス意識の高いソリューション
- モバイルアプリのアクセシビリティベストプラクティス
- 適切なエラーハンドリングとエッジケース管理
- 必要に応じて適切なコメントを含む明確で自己文書化されたコード
