# レビュー対応

レビュー結果: $ARGUMENTS

## 実行内容

### Step 1: レビューコメントの収集

- `$ARGUMENTS` が指定されている場合はそのファイル/内容を参照
- 指定がない場合は会話中の直前のレビュー結果（`/review` または `/review-parallel` の出力）を使用

### Step 2: 各コメントの評価

| 判断 | 基準 |
|------|------|
| **Fix** | バグ・セキュリティ問題・明らかな改善 |
| **Skip** | 好み・スタイルの問題・プロジェクト方針に合わない |
| **Discuss** | 大きな設計変更を伴う・要件の確認が必要 |

各コメントに対して判断と理由を一覧で示す：

```markdown
| # | コメント要旨 | 判断 | 理由 |
|---|------------|------|------|
| 1 | ...        | Fix  | バグの可能性があるため |
| 2 | ...        | Skip | 個人好みのスタイル指摘 |
| 3 | ...        | Discuss | アーキテクチャ変更を要するため |
```

### Step 3: Fix 対象の実装

評価表を示した後、ユーザーの確認を得てから実装する。

**実装方針:**
- Fix と判断したものをすべて実装
- 1コメント = 最小限の変更（関係ない箇所は触らない）
- 実装後は lint/型チェックが通ることを確認

### Step 4: 完了報告

```markdown
## 対応完了

### 実施した修正
- [ ] #1: [コメント要旨] → [実施内容]
- [ ] #3: [コメント要旨] → [実施内容]

### スキップしたコメント
- #2: [理由]

### 要相談
- #4: [論点]
```
