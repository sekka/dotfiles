# Figma実装依頼ガイド

精度の高い実装に必要な情報をまとめた依頼者向けガイド。

---

## 必須情報

### 1. FigmaのURL または ノードID

```
# フレーム全体
https://www.figma.com/design/XXXXX/ProjectName?node-id=1234-5678

# 特定コンポーネント指定
node-id=1234-5678 のButtonコンポーネント
```

URLがない場合は「どのフレームか」を明示的に説明する。

### 2. 実装先の技術スタック

```
# 良い例
React + Tailwind CSS v4
Vue 3 + Panda CSS
Next.js 15 + shadcn/ui

# 悪い例
フロントエンドで実装して
```

### 3. Design Systemの情報

```
# トークンファイルを渡す
tokens.json / tailwind.config.ts を渡す

# またはルールを明示する
色は必ずCSS変数（--color-primary-500）を使う
スペーシングはTailwindクラスのみ（px直書き禁止）
```

### 4. 実装範囲の明示

```
# 良い例
ログイン画面のCardコンポーネントのみ（モーダルは対象外）
スマホ版のみ。デスクトップ対応は不要

# 悪い例
この画面を実装して（範囲が広すぎる）
```

---

## あると精度が上がる情報

| 情報 | 効果 |
|------|------|
| インタラクション仕様（hover/active状態） | デザインに書いてない動きを補完できる |
| 既存コンポーネントのコード | 既存パターンに合わせて実装できる |
| 「PixelPerfect不要、意図重視」などの方針 | 細部の判断基準が明確になる |
| フォントのインポート方法（Google Fonts / ローカル） | フォントが正確に反映される |

## 依頼文テンプレート

```
Figmaのデザイン通りに実装してください。

URL: https://www.figma.com/design/XXXXX?node-id=1234-5678
対象: [コンポーネント名 or 画面名]
スタック: [フレームワーク + CSSライブラリ]
スタイル方針: [例: Tailwindクラスのみ使用（インラインstyle禁止）]
Design Token: [例: tailwind.config.ts の colors を使う]

スコープ:
- [含める範囲]
- [除外する範囲]

既存コンポーネント: [参考にすべきファイルパス]
```

---

## Figma MCP の有無による対応

| 状況 | 推奨 |
|------|------|
| Figma MCPあり | URLとnode-idを渡すだけでOK。自動取得される |
| MCPなし | デザインのスクリーンショット + FigmaのInspectパネルの数値を貼り付け |
| MCPなし（簡易） | スクリーンショットのみ貼り付けて「このデザインを実装して」でも可 |
