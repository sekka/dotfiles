# PDF画像からMarkdown抽出プロンプト

以下の画像をMarkdown形式に変換してください。

## 変換ルール

### 1. 見出し

視覚的な階層に基づいて見出しレベル（H1〜H6）を適用してください。

- **H1 (`#`)**: ページタイトル、章タイトル
- **H2 (`##`)**: セクションタイトル
- **H3 (`###`)**: サブセクション
- **H4〜H6**: より細かい階層

**例:**

```markdown
# 第1章 概要

## 1.1 背景

### 1.1.1 現状の課題
```

### 2. 表

Markdownテーブル形式で正確に再現してください。

- セルの結合は `colspan` / `rowspan` を使わず、可能な限り分割
- 数値は右揃え、テキストは左揃え
- 表のキャプションは表の直前に記載

**例:**

```markdown
| 項目       | 2023年 | 2024年 |
|-----------|--------|--------|
| 売上高     | 100億円 | 120億円 |
| 営業利益   | 10億円  | 15億円  |
```

### 3. リスト

箇条書きと番号付きリストを適切に使用してください。

- **箇条書き (`-`)**: 順序が重要でない場合
- **番号付きリスト (`1.`)**: 順序が重要な場合、手順

**例:**

```markdown
- 項目1
- 項目2
  - サブ項目2-1
  - サブ項目2-2

1. ステップ1
2. ステップ2
3. ステップ3
```

### 4. グラフ・図

画像やグラフは説明文で記載してください。

**形式:**

```markdown
[図: 〇〇を示すグラフ]

（可能であれば、グラフの主要なデータポイントを箇条書きで補足）

- 2023年: 100万件
- 2024年: 150万件
- 2025年予測: 200万件
```

### 5. 脚注・注釈

脚注や注釈は、そのセクションの末尾にまとめてください。

**例:**

```markdown
## セクションタイトル

本文[^1]

[^1]: これは脚注です。
```

### 6. 引用

引用は `>` を使用してください。

**例:**

```markdown
> これは引用文です。
> 複数行にわたる場合もあります。
```

### 7. コード・数式

- **インラインコード**: バッククォート `` ` `` で囲む
- **コードブロック**: トリプルバッククォート `` ``` `` で囲む
- **数式**: LaTeX形式（`$...$` または `$$...$$`）

**例:**

```markdown
変数 `x` を使用します。

\`\`\`python
def hello():
    print("Hello, World!")
\`\`\`

数式: $E = mc^2$
```

### 8. 強調

- **太字**: `**テキスト**`
- **斜体**: `*テキスト*`
- **取り消し線**: `~~テキスト~~`

### 9. リンク

URLが記載されている場合はリンク形式で記載してください。

**例:**

```markdown
[リンクテキスト](https://example.com)
```

### 10. 不要な要素の除去

以下の要素は出力しないでください:

- フロントマター（YAML形式のメタデータ）
- 過剰な空行（連続する空行は1行に）
- ページ番号（必要に応じてコメントで記載）
- ヘッダー・フッター（繰り返し表示される要素）

## 出力形式

- **文字コード**: UTF-8
- **改行コード**: LF（`\n`）
- **空行**: セクション間に1行の空行
- **言語**: 元のPDFの言語を保持（日本語/英語混在可）

## 品質基準

### 構造の正確性

- 見出し階層が視覚的に正確
- 表の内容が正確に再現されている
- リストの階層が正確

### テキストの正確性

- 誤字脱字がない
- 数値が正確
- 固有名詞が正確

### 可読性

- 適切な空行配置
- 長すぎる行は適宜改行（80〜100文字目安）
- 視覚的に読みやすい

## 特殊ケース

### 複数カラムレイアウト

左カラムから順に読み、右カラムに移動してください。

### スキャン画像の場合

- OCRの精度が低い場合は、可能な範囲で補正
- 判読不能な部分は `[判読不能]` と記載

### 日本語・英語混在

- 適切に言語を切り替え
- 英単語の前後に半角スペースを入れない（日本語の場合）

**例:**

```markdown
これはMarkdown形式です。
```

## 検証

変換後、以下を確認してください:

- [ ] 見出し階層が正確
- [ ] 表が正確に再現されている
- [ ] リストが正確
- [ ] 図の説明が適切
- [ ] 不要な要素が除去されている
- [ ] 誤字脱字がない
