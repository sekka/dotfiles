# ビジネスコミュニケーション作成

## 概要

ビジネスコミュニケーション、ユーザーサポート、チーム共有、アナウンス作成を包括的に支援するスキルです。

## 実行フロー

### Step 1: ユーザーサポート対応

#### HEARD Framework

```
H: Hear（聞く） - ユーザーの声を受け止め、問題を理解
E: Empathize（共感） - ユーザーの立場に立ち、感情に寄り添う
A: Apologize（謝罪） - 必要に応じて誠実に謝罪
R: Resolve（解決） - 具体的な解決策を提示
D: Delight（喜ばせる） - 期待以上の対応で満足度向上
```

#### 状況別対応パターン

- バグ報告への返信
- 機能リクエストへの返信
- 苦情への返信

### Step 2: チームコミュニケーション

#### BLUF原則（Bottom Line Up Front）

```
【結論】1-2文で要点
【詳細】背景や詳細情報
【アクション】次に必要なこと
```

#### 進捗共有テンプレート

- サマリー
- 完了した項目
- 進行中のタスク
- ブロッカー/課題
- 次週の予定
- メトリクス

### Step 3: インシデントコミュニケーション

- 初動連絡: 影響範囲、現在の状況
- 中間報告: 対応経過、次のステップ
- 事後報告（ポストモーテム）: 根本原因、恒久対策、学び

### Step 4: リクエストとリマインド

- 依頼メッセージ: 背景、期限、成果物
- リマインドメッセージ: 進捗確認、サポート提案

### Step 5: ステークホルダー別コミュニケーション

| 対象     | トーン     | 詳細度     | フォーカス         |
| -------- | ---------- | ---------- | ------------------ |
| 経営層   | フォーマル | 簡潔       | インパクト、リスク |
| 同チーム | カジュアル | 詳細OK     | タスク、技術       |
| 他チーム | 丁寧       | 中程度     | 依存関係、期待     |
| 外部     | フォーマル | 必要最小限 | 結論、約束         |

## 出力成果物

1. **サポート返信**: 問い合わせへの回答文
2. **共有メッセージ**: 進捗報告、決定事項の共有
3. **アナウンス**: 障害通知、リリース告知
4. **議事録/要約**: ミーティングノート
5. **リマインド**: フォローアップ、期限リマインド
6. **エグゼクティブサマリー**: 経営層向け報告

## ベストプラクティス

1. **明確性**: 曖昧さを排除、5W1Hを意識
2. **簡潔性**: 無駄な言葉を削る、箇条書きを活用
3. **行動可能性**: 次のアクションを明確に、担当と期限を明記
4. **適切なトーン**: 相手と状況に合わせる、共感を示す
5. **タイムリー性**: 迅速に返信、定期的なアップデート
