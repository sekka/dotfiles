```bash

| 変数名 | 説明 | 例 |

```bash
npm run db:migrate
npm run db:seed
```

## トラブルシューティング

### `EACCES: permission denied`

**原因**: npm グローバルディレクトリの権限問題

**解決策**:

```bash
sudo chown -R $(whoami) ~/.npm
```

### `Module not found`

**原因**: 依存関係のインストール不完全

**解決策**:

```bash
rm -rf node_modules package-lock.json
npm install
```

---

## 認証

### POST /auth/login

ユーザー認証を行い、アクセストークンを取得します。

**リクエスト**

```json
{
  "email": "user@example.com",
  "password": "password123"
}
```

**レスポンス**

```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIs...",
  "token_type": "Bearer",
  "expires_in": 3600
}
```

**エラーレスポンス**

| ステータス | 説明 |
|-----------|------|
| 400 | リクエスト形式エラー |
| 401 | 認証失敗 |
| 429 | レートリミット超過 |

---

## ユーザー

### GET /users/{id}

指定されたIDのユーザー情報を取得します。

**パスパラメータ**

| パラメータ | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| id | string | Yes | ユーザーID |

**レスポンス**

```json
{
  "id": "user_123",
  "email": "user@example.com",
  "name": "田中太郎",
  "created_at": "2024-01-01T00:00:00Z"
}
```

### POST /users

新しいユーザーを作成します。

**リクエストボディ**

| フィールド | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| email | string | Yes | メールアドレス |
| name | string | Yes | 表示名 |
| password | string | Yes | パスワード（8文字以上） |

```

## データフロー

### ユーザー認証フロー

1. ユーザーがログインフォームを送信
2. API Gatewayで認証リクエストを受信
3. Auth Serviceでクレデンシャルを検証
4. JWTトークンを生成して返却
5. クライアントがトークンをローカルストレージに保存
