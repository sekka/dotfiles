# ドキュメント作成

## Quick Start

1. ドキュメントの種類と対象読者を確認
2. 適切なテンプレートを選択
3. 品質チェックリストで検証

## ワークフロー

### Step 1: 要件確認

- [ ] ドキュメントの種類（セットアップ/マニュアル/手順書/FAQ）
- [ ] 対象読者（ユーザー/開発者/両方）
- [ ] 前提知識のレベル

### Step 2: 構成設計

- [ ] 目的を明確化
- [ ] 論理的な順序で構成
- [ ] 必要なセクションを洗い出し

### Step 3: 執筆

- [ ] テンプレートに沿って記述
- [ ] 具体的で再現可能な手順
- [ ] コピー可能なコマンド/コード

### Step 4: 検証

- [ ] 品質チェックリストで確認
- [ ] 実際に手順を実行して検証

### FAQ

```markdown
# よくある質問

## [カテゴリ]

### Q: [質問]
A: [回答]

### Q: [質問]
A: [回答]
````

## 品質チェックリスト

- [ ] 目的が冒頭で明確に述べられている
- [ ] 前提条件が漏れなく記載されている
- [ ] 手順が具体的で再現可能
- [ ] コマンド/コードがコピー可能
- [ ] エラー時の対処法がある
- [ ] 専門用語は初出時に説明されている

## 執筆ガイドライン

### 読者中心の設計

- 読者が何を達成したいかを起点に構成
- 前提知識に合わせた説明レベル
- 不要な情報は省略

### 実用性の重視

- 抽象的な説明より具体例
- 実際のエラーメッセージと対処法
- 「なぜ」より「どうやって」を優先
