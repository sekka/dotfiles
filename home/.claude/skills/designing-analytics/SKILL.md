# アナリティクス設計とデータ可視化

## 概要

データ分析、アナリティクス設計、可視化、ダッシュボード設計、レポート作成を包括的に支援するスキルです。

## 実行フロー

### Step 1: 計測戦略の設計

#### イベント計測設計

**イベントタイプ**

```
ユーザーアクション
├── ページビュー: 画面表示
├── クリック: ボタン/リンク操作
├── フォーム: 入力/送信
├── 機能利用: 特定機能の使用
└── エンゲージメント: スクロール/滞在時間

コンバージョン
├── サインアップ
├── 購入/課金
├── 目標達成
└── リテンション指標
```

**イベントスキーマ例**

```json
{
  "event_name": "feature_used",
  "properties": {
    "feature_name": "export",
    "format": "csv",
    "user_tier": "pro"
  },
  "context": {
    "user_id": "xxx",
    "session_id": "xxx",
    "timestamp": "2025-01-01T00:00:00Z"
  }
}
```

#### ファネル分析設計

```
認知 → 獲得 → アクティベーション → リテンション → 収益 → 紹介
  │       │           │               │           │       │
  │       │           │               │           │       └─ 紹介率
  │       │           │               │           └─ ARPU, LTV
  │       │           │               └─ D1/D7/D30リテンション
  │       │           └─ アクティベーション率
  │       └─ CVR（訪問→登録）
  └─ トラフィック、インプレッション
```

#### コホート分析

| 分析タイプ   | 目的                            |
| ------------ | ------------------------------- |
| 時間コホート | 登録週/月ごとのリテンション比較 |
| 行動コホート | 特定行動を取ったユーザーの追跡  |
| 機能コホート | 機能利用者 vs 非利用者の比較    |

### Step 2: ダッシュボード設計

#### ダッシュボード階層

```
レベル1: エグゼクティブ
├── 主要KPI（3-5個）
├── トレンドサマリー
└── アラート/異常値

レベル2: マネージャー
├── 部門別指標
├── 目標vs実績
└── ドリルダウン

レベル3: アナリスト
├── 詳細データ
├── セグメント分析
└── カスタムフィルタ
```

#### レイアウト原則

- 左上から右下へ重要度を配置
- 最重要KPIは大きく、目立つ位置に
- 関連する情報はグループ化

### Step 3: データ可視化設計

#### チャート選定ガイド

| 目的               | 推奨チャート           |
| ------------------ | ---------------------- |
| 比較（カテゴリ間） | 棒グラフ、グループ棒   |
| 比較（時系列）     | 折れ線グラフ           |
| 構成比             | 円グラフ、積み上げ棒   |
| 分布               | ヒストグラム、箱ひげ図 |
| 相関               | 散布図                 |
| ファネル           | ファネルチャート       |

#### アクセシビリティ

- 色だけに頼らない（パターン、形状を併用）
- 十分なコントラスト比
- スクリーンリーダー対応

### Step 4: レポート作成

```markdown
## エグゼクティブサマリー
- 主要な発見3点
- 重要な変化点

## KPI概要
| 指標 | 今期 | 前期 | 変化 |

## 詳細分析
### [セクション1]
- トレンド分析
- 要因分析
- インサイト

## 推奨アクション
| アクション | 優先度 | 担当 | 期限 |
```

### Step 5: 主要指標（KPI）の定義

#### 成長指標

- ユーザー数（MAU/DAU）
- 成長率（MoM/YoY）
- 獲得チャネル別パフォーマンス

#### エンゲージメント指標

- DAU/MAU比率（Stickiness）
- セッション長/頻度
- 機能利用率

#### リテンション指標

- D1/D7/D30リテンション
- チャーン率
- リテンションカーブ

#### 収益指標

- MRR/ARR
- ARPU/ARPPU
- LTV/CAC比率

## 出力成果物

1. **計測戦略書**: イベント定義、ファネル設計
2. **KPI定義書**: 指標定義、算出式、目標値
3. **ダッシュボード設計書**: レイアウト、カード構成
4. **可視化仕様書**: チャート選定、配色
5. **レポートテンプレート**: 定期レポートの構成

## ベストプラクティス

1. **意思決定に直結する指標を優先**: Vanity Metricsを避ける
2. **シンプルさを保つ**: 主要KPIは3-5個に絞る
3. **定期的なレビューと改善**: 四半期ごとにKPIを見直す
4. **プライバシーとセキュリティ**: 最小限のデータ収集
5. **データ品質の確保**: 定義を文書化
