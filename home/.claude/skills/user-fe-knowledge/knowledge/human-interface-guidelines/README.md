# Human Interface Guidelines (HIG) ナレッジベース

Apple Human Interface Guidelines (HIG) のリファレンスとして、主要な概念とベストプラクティスをまとめたナレッジベースです。

## 概要

Human Interface Guidelinesは、Apple プラットフォーム（iOS、iPadOS、macOS、tvOS、watchOS、visionOS）向けのアプリやゲームを設計する際の指針を提供します。

**公式URL**: https://developer.apple.com/design/human-interface-guidelines/
**日本語版**: https://developer.apple.com/jp/design/human-interface-guidelines/

## 重要な注意事項

⚠️ **このナレッジベースの制限について**

Apple HIG公式サイトはJavaScriptで動的にコンテンツをレンダリングするため、自動抽出が困難です。このナレッジベースは以下のように構成されています：

- **ディレクトリ構造**: HIG公式の構造を反映
- **概要とテンプレート**: 各セクションの概要と主要トピック
- **公式リンク**: 詳細は公式ドキュメントへのリンクを提供
- **手動追記**: 重要な内容は手動で追記する想定

## デザイン原則（3つの柱）

Apple HIGの基本原則：

### 1. Hierarchy（階層性）
コントロールとインターフェース要素が明確な視覚的階層を確立し、下層のコンテンツを際立たせる

### 2. Harmony（調和）
ハードウェアとソフトウェアの同心円デザインに合わせ、インターフェース要素、システム体験、デバイス間の調和を創出

### 3. Consistency（一貫性）
プラットフォームの慣習を採用し、ウィンドウサイズやディスプレイに継続的に適応する一貫したデザインを維持
