# UI Design Usability Checklist: 50 Essential Rules from Real Projects

Nielsen's 10 usability heuristics applied to real client projects. 50 practical rules with downloadable checklist.

## 1. System Status Visibility (10 principles)
- Multi-step forms: Break lengthy forms into sections with progress indicators
- Mode indication: Clearly show when users are in a special mode (e.g., "〇〇中" status bar)
- Transition animations: Use motion to help users understand screen changes and system structure
- Modal backgrounds: Transparently dim backgrounds to clarify modal vs. full page transitions
- Filter indicators: Display active filter counts, color changes, or condition chips
- Empty states: Provide guidance when no content exists yet
- Proximity feedback: Show status changes near the triggering action
- Navigation clarity: Use titles and breadcrumbs to establish user location
- Toggle clarity: Add visual elements to distinguish current state from post-action state

## 2. System-World Alignment (7 principles)
- User language: Avoid programmer terminology; use explanations users understand
- Button clarity: State what happens when clicked, not generic terms like "OK"
- Verb-based labels: Use action words ("Save," "Delete") over neutrals like "Yes"
- Date formatting: Use standard separators (/, -, etc.) and zero-padding for consistency
- Users need "understandable language" rather than "system error messages"

## 3. User Control & Freedom (4 principles)
- Enable quick error correction and action reversal
- Carousel controls: Combine dot indicators with arrow buttons for easier navigation
- Scroll hints: Partially reveal off-screen content to encourage exploration
- Clickability signals: Add shadows, arrows, or visual depth to indicate interactive cards/lists

## 4. Consistency & Standards (7 principles)
- Title alignment: Match button text to destination page titles
- Tab appearance: Use distinct visual separation with parallel layout
- Visual distinction: Don't use identical styling for different functions
- Color-function matching: Red for danger/deletion, green for safe/complete states
- Link vs. button: Links enable navigation without data changes; buttons trigger actions
- Text links: Include underlines and distinct colors (preferably blue)
- External links: Add icons to distinguish new tabs or external sites

## 5. Error Prevention (7 principles)
- Tap spacing: Maintain sufficient gaps between interactive elements
- Dangerous actions: Isolate deletion/critical options away from standard content
- Larger targets: Make clickable areas generous beyond visual boundaries
- Early conditions: Present validation requirements before users complete input
- Scroll preservation: Keep critical info visible during horizontal scrolling
- Real-time validation: Show which conditions are met as users type
- Constrained input: Use pickers, sliders, and dropdowns instead of free-text entry

## 6. Recognition Over Recall (3 principles)
- Autocomplete: Show suggestions during typing to reduce manual input errors
- Placeholders: Provide input examples alongside labels (not as replacements)
- Form width: Adjust field widths to hint at expected content type

## 7. Flexibility & Efficiency (5 principles)
- Accordions: Reduce scrolling burden by collapsing lengthy content
- Smart defaults: Pre-select the most likely choice for most users
- Auto-fill: Automatically populate fields derivable from existing data
- Radio buttons: Use for 2-3 options; save dropdowns for longer lists
- Toggle distinction: Use switches for immediate changes; checkboxes for batch modifications

## 8. Aesthetic & Minimalist Design (8 principles)
- Line spacing: 0.5–1 character height between lines for readability
- Line length: 25–40 characters per line prevents reading fatigue
- Visual adjustment: Compensate for optical illusions (e.g., visual vs. mathematical centering)
- Minimize emphasis: Avoid overusing bold, color, or decorative attributes
- Transparent information: Show features directly rather than hiding behind hover states
- Button hierarchy: Vary appearance to distinguish primary, secondary, and tertiary actions
- Form grouping: Organize related fields into clear sections
- Contrast standards: Maintain 4.5:1 minimum contrast ratio; avoid color-only distinction
- Cognitive load: Limit distinct information chunks to 7±2 items

## 9. Error Recovery (1 principle)
- Inline errors: Display validation messages next to problematic fields, not at page top

## 10. Help & Documentation (1 principle)
- Icon labels: Always pair icons with text; use tooltips for standalone icons
