# creating-github-issues

技術調査の結果や会話内容を、構造化された GitHub/GitLab Issue に変換するスキル。

## 特徴

- **テンプレート形式**: タイトル、概要、詳細、完了条件、実装上の注意点、ブランチ名を含む
- **散文的な説明**: 箇条書き過多を避け、流れのある文章で記述
- **完了条件のチェックリスト**: 検証可能な条件を列挙
- **ブランチ名提案**: Git Flow に準拠したブランチ名を自動生成

## 使い方

### 基本的な使用例

```
認証トークンの有効期限切れ時のエラーハンドリングを改善したい。
これを GitHub Issue にまとめて。
```

### 出力例

```markdown
# [Fix] 認証トークン有効期限切れ時のエラーハンドリング

## 概要
現在、認証トークンの有効期限が切れると 401 エラーが返されるが、
フロントエンドでのエラーハンドリングが不十分...

## 詳細
...

## 完了条件
- [ ] Axios インターセプターで 401 エラーをハンドリング
- [ ] リフレッシュトークンによる自動再ログイン機能の実装
...

## 実装上の注意点
...

## 対応ブランチ名
fix/auth-token-expiry-handling
```

## トリガーキーワード

- 「Issue を作成して」
- 「タスクとして記録して」
- 「GitHub Issue にして」
- 「これを Issue にまとめて」
