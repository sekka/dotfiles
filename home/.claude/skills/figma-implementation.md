# Figma 実装フロー

目的: Figma スタイルリサーチで得た情報を基に、既存コードベースへ正確に落とし込む。

## 基本方針
- 先に figma-style スキルで取得した実測データを前提にする。足りない場合は Figma MCP を追加で呼び出して補う。
- 既存コンポーネントと設計体系（デザインシステム、トークン、命名）を優先的に再利用する。
- 状態（default/hover/focus/active/disabled）、バリアント、レスポンシブの挙動をコードに反映する。
- 変更範囲を最小化し、影響範囲が広い場合は理由とリスクを明示する。

## 進め方
1. 前提確認: 対象コンポーネント/画面、優先デバイス、テーマの有無、動作環境（ライブラリやフレームワーク）を把握する。
2. 設計: 既存のレイアウト/コンポーネント構造に合わせて props・state・variant を整理し、必要に応じて小さい単位に分割する。
3. 実装: リポジトリのスタイルシステム（例: design tokens, CSS Modules, Tailwind など）に沿って数値とトークンを適用し、アクセシビリティ（セマンティクス/ARIA/tab 移動）を確保する。
4. 検証: 主要状態とブレークポイントで見た目と挙動を確認し、必要なビルド/テスト/リントを実行する。
5. 共有: 変更点とテスト結果を簡潔にまとめ、残課題や追加で必要な Figma 情報があれば明示する。
