# AIエージェントへの効果的な指示パターン

## 複雑タスクは「手順書 + スクリプト先行」方式で依頼する

> 出典: https://x.com/suthio_/status/1939953298355728665
> 投稿日: 2025-07-01
> 追加日: 2026-04-13

複雑な作業を AIエージェントに依頼する際、「○○をお願いします」と直接頼むよりも、
まず「○○を行うための手順書とスクリプトを用意してください」と指示し、
次のセッションでその手順書をもとにスクリプトを実行してもらう方が出力の質が上がる。

### パターン

```
# セッション 1
「○○を行うための手順書とスクリプトを用意してください」
→ エージェントが手順書・スクリプトを作成

# セッション 2
「用意した手順書に従って、スクリプトを実行してください」
→ より正確・安定した実行
```

### ユースケース
- データベースマイグレーションなど、手順が複雑な操作
- 複数ファイルにまたがるリファクタリング
- デプロイ・インフラ変更など、やり直しが効かない作業
- 長い実装タスクをセッションをまたいで行う場合

### なぜ効果的か
- エージェントが実行前に全体像を整理するため、抜け漏れが減る
- 手順書がある状態での実行は「指示に従う」モードになり出力が安定する
- セッションが切れても手順書さえあれば再開できる

## バグ修正時は「console.log でステップバイステップ」と明示する

> 出典: https://x.com/igz0/status/1894336807711772997
> 投稿日: 2025-02-25
> 追加日: 2026-04-13

AIエージェントに重いバグ修正を頼む際、同じエラーで何度も詰まることがある。
バグ修正を依頼するとき「console.log を仕込んでステップバイステップで原因究明に努めてください」と指示すると問題解決の質が大きく向上する。

### 指示テンプレート

```
「○○のバグを修正してください。
 console.log を仕込んでステップバイステップで原因究明に努めてください。」
```

### ユースケース
- 原因が不明な再現バグ
- 複数のコンポーネントにまたがる問題
- 非同期処理・タイミング系のバグ
- エラーメッセージが抽象的で手がかりが少ない場合

### なぜ効果的か
- エージェントが仮説を立てながら段階的に絞り込む
- ログ出力を確認するため、見当違いの修正を繰り返すことが減る
- デバッグ過程が可視化され、レビューしやすくなる
