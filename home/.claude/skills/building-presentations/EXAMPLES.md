## 例1: 新規プロダクト提案（問題解決型）

### プレゼン概要

- **目的**: 社内承認を得る
- **聴衆**: 経営層（非技術者含む）
- **時間**: 15分
- **核心メッセージ**: AIチャットボットで顧客サポートコストを30%削減できる

### ストーリーライン

1. [導入] 顧客サポートのコスト増加問題
2. [展開] AIチャットボットによる自動化提案
3. [結論] 投資回収とアクション

---

### スライド1: タイトル

**見出し**: 顧客サポート業務の効率化提案

**本文**: AIチャットボット導入によるコスト削減プラン

**ビジュアル案**: シンプルなタイトルスライド

**ノート**: 本日は顧客サポート業務の効率化について、具体的な改善提案をさせていただきます。

---

### スライド2: 現状の課題

**見出し**: 顧客サポートコストが年間20%増加

**本文**:

- 月間問い合わせ件数: 5,000件（前年比+25%）
- サポート人員: 15名（限界に達している）
- 平均対応時間: 15分/件
- 年間コスト: 4,800万円

**ビジュアル案**: 問い合わせ件数の推移グラフ（右肩上がり）

**ノート**: 事業成長に伴い、顧客からの問い合わせが急増しています。現在のチームでは対応が追いつかず、応答時間も長期化しています。

---

### スライド3: なぜ解決が難しいのか

**見出し**: 単純な人員増強では根本解決にならない

**本文**:

- 採用コスト: 1名あたり年間400万円
- 育成期間: 3ヶ月
- 離職率: 年間30%（業界平均）
- スケーラビリティの限界

**ビジュアル案**: コスト増加の悪循環を示す図

**ノート**: 人を増やしても、育成コストと離職率の高さから、持続的な解決策にはなりません。

---

### スライド4: 提案: AIチャットボット導入

**見出し**: よくある質問の80%を自動化

**本文**:

- 過去の問い合わせデータを学習
- 24時間365日対応可能
- 複雑な問題は人間にエスカレーション
- 段階的に導入（3ヶ月で全展開）

**ビジュアル案**: チャットボットのフロー図

**ノート**: 簡単な問い合わせはAIが即座に回答し、複雑な案件だけを人間が対応する仕組みです。

---

### スライド5: 期待される効果

**見出し**: 年間1,500万円のコスト削減

**本文**:

- 自動化率: 80%（4,000件/月）
- 対応時間短縮: 平均15分 → 2分
- 人員最適化: 15名 → 10名
- 顧客満足度向上: 即時対応による

**ビジュアル案**: コスト削減額を示す棒グラフ

**ノート**: 初期投資は12ヶ月で回収でき、以降は継続的にコスト削減効果が得られます。

---

### スライド6: 実装プラン

**見出し**: 3ヶ月で段階的に展開

**本文**:

| フェーズ | 期間 | 内容 |
|---------|------|------|
| Phase 1 | 1ヶ月 | データ準備・学習 |
| Phase 2 | 1ヶ月 | パイロット運用（20%のトラフィック） |
| Phase 3 | 1ヶ月 | 全面展開・チューニング |

**ビジュアル案**: タイムライン図

**ノート**: リスクを最小化するため、段階的に導入します。各フェーズで効果を検証しながら進めます。

---

### スライド7: 次のステップ

**見出し**: 今月中に承認をいただきたい

**本文**:

- **必要な投資額**: 初期500万円 + 運用100万円/年
- **期待ROI**: 12ヶ月で投資回収
- **意思決定期限**: 12月末
- **開始予定**: 1月第1週

**ビジュアル案**: CTAボタン風のデザイン

**ノート**: 来年度の予算計画に反映するため、今月中にご承認をいただけますでしょうか。

---

### Q&A想定

- **Q**: 既存のサポートメンバーの雇用はどうなるのか？
- **A**: 削減ではなく、より高度な対応や顧客フォローに注力していただきます。自然減での調整を想定しています。

- **Q**: AIが間違った回答をするリスクは？
- **A**: 初期段階では人間が回答を確認し、段階的に自動化率を上げます。また、不確実な場合は自動的に人間にエスカレーションする仕組みを実装します。

## 例3: 技術勉強会（教育型・30分）

### プレゼン概要

- **目的**: React Hooksの基本を理解してもらう
- **聴衆**: 開発チームメンバー（React初心者含む）
- **時間**: 30分
- **核心メッセージ**: Hooksで関数コンポーネントが強力になる

---

### スライド1: タイトル

**見出し**: React Hooks 入門

**本文**: 関数コンポーネントでステート管理

**ビジュアル案**: Reactロゴ

**ノート**: 今日はReact Hooksの基本を学び、実際に使えるようになることが目標です。

---

### スライド2: なぜHooksか

**見出し**: クラスコンポーネントの課題を解決

**本文**:

- ライフサイクルメソッドの複雑さ
- ロジックの再利用が困難
- `this`の扱いが難解

**ビジュアル案**: クラスコンポーネントの複雑なコード例

**ノート**: Hooksはこれらの課題を解決する新しいアプローチです。

---

### スライド3: Hooksとは

**見出し**: 関数コンポーネントにステートとライフサイクルを追加

**本文**:

- `useState`: ステート管理
- `useEffect`: 副作用の処理
- `useContext`: コンテキストの利用
- カスタムHooks: ロジックの再利用

**ビジュアル案**: Hooksの種類を示す図

**ノート**: まずは基本的な2つ（useState、useEffect）を学びます。

---

### スライド4: useState の使い方

**見出し**: ステート管理がシンプルに

**本文**:

```jsx
const [count, setCount] = useState(0);

return (
  <button onClick={() => setCount(count + 1)}>
    Count: {count}
  </button>
);
```

**ビジュアル案**: コード例

**ノート**: デストラクチャリングで値と更新関数を取得します。

---

### スライド5: useEffect の使い方

**見出し**: 副作用を宣言的に扱う

**本文**:

```jsx
useEffect(() => {
  document.title = `Count: ${count}`;

  return () => {
    // クリーンアップ
  };
}, [count]); // 依存配列
```

**ビジュアル案**: コード例

**ノート**: 第2引数の依存配列が重要です。

---

### スライド6: ハンズオン

**見出し**: 実際に書いてみよう

**本文**:
課題: カウンターアプリを作成

1. `useState`でカウント状態を管理
2. `useEffect`でタイトルを更新
3. インクリメント/デクリメントボタンを実装

**ビジュアル案**: 完成イメージ

**ノート**: 15分間、実際にコードを書いてみてください。わからないことがあれば声をかけてください。

---

### スライド7: まとめ

**見出し**: Hooksで関数コンポーネントが強力に

**本文**:

- ✅ `useState`: ステート管理
- ✅ `useEffect`: 副作用の処理
- ✅ カスタムHooks: ロジックの再利用

次に学ぶべきこと:

- `useMemo`, `useCallback` (パフォーマンス最適化)
- カスタムHooksの作成

**ビジュアル案**: チェックリスト

**ノート**: 質問があれば遠慮なくどうぞ！
