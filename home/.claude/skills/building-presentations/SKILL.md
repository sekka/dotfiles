# プレゼンテーション作成

## Quick Start

1. 目的と聴衆を確認
2. ストーリーラインを設計
3. スライド構成と原稿を作成

## ワークフロー

### Step 1: 要件確認

- [ ] プレゼンの目的（提案/報告/教育/ピッチ）
- [ ] 聴衆（経営層/技術者/顧客/投資家）
- [ ] 時間（5分/15分/30分）
- [ ] 形式（対面/オンライン/録画）

### Step 2: ストーリーライン設計

- [ ] 核心メッセージを1文で定義
- [ ] 聴衆の課題/関心を特定
- [ ] 論理の流れを決定

### Step 3: スライド構成

- [ ] 各スライドの見出しを作成
- [ ] 本文/箇条書きを記述
- [ ] 図解の説明を追加

### Step 4: 仕上げ

- [ ] CTA/まとめスライドを作成
- [ ] 時間配分を確認
- [ ] Q&A想定を準備

## ストーリー構成パターン

### 問題解決型（提案/ピッチ向け）

1. **現状**: 聴衆が直面している課題
2. **問題**: なぜ解決が難しいか
3. **解決策**: 提案する解決方法
4. **根拠**: なぜ効果があるか（データ/事例）
5. **行動**: 次のステップ

### 成果報告型（報告向け）

1. **目的**: 何を達成しようとしたか
2. **アプローチ**: どう取り組んだか
3. **結果**: 何が得られたか
4. **学び**: 次に活かせること
5. **次のアクション**: 今後の予定

### 教育型（ワークショップ向け）

1. **なぜ**: このトピックが重要な理由
2. **何**: 核心概念の説明
3. **どうやって**: 具体的な方法/手順
4. **実践**: ハンズオン/演習
5. **まとめ**: 要点の復習

## スライド設計ガイドライン

### 1スライド1メッセージ

- 見出しで結論を述べる
- 本文は補足情報のみ
- 箇条書きは3〜5項目まで

### 時間配分の目安

| プレゼン時間 | スライド枚数 | 1枚あたり |
| ------------ | ------------ | --------- |
| 5分          | 5〜7枚       | 45〜60秒  |
| 15分         | 12〜15枚     | 60〜75秒  |
| 30分         | 20〜25枚     | 70〜90秒  |

## 出力フォーマット

```markdown
- **聴衆**: [誰に向けて]
- **時間**: [X分]
- **核心メッセージ**: [最も伝えたいこと]

## ストーリーライン

1. [導入] → 2. [展開] → 3. [結論]

### スライド2: [見出し]

...

## Q&A想定

- **Q**: [想定質問]
- **A**: [回答案]
```
