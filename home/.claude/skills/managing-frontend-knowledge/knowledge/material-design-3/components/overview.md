# Components Overview（コンポーネント概要）

Material Design 3は、30以上の再利用可能なUIコンポーネントを提供しています。このドキュメントでは、主要コンポーネントの使用時期と選択基準を解説します。

## コンポーネント選択のフレームワーク

### 1. ユーザーの意図を理解する

コンポーネントを選ぶ前に、ユーザーが何をしようとしているかを明確にします：

- **アクション**: ボタン、FAB
- **ナビゲーション**: Navigation Bar、Drawer、Tabs
- **情報表示**: Cards、Lists
- **データ入力**: Text Fields、Checkboxes、Chips
- **フィードバック**: Snackbars、Dialogs
- **選択**: Menus、Dropdowns

### 2. コンテキストを考慮する

- **デバイスサイズ**: モバイルかデスクトップか
- **画面の役割**: ホーム、詳細、設定など
- **情報の複雑さ**: シンプルか複雑か
- **ユーザーの習熟度**: 初心者か熟練者か

### 3. アクセシビリティを優先する

- すべてのユーザーが操作できるか
- キーボードナビゲーションに対応しているか
- スクリーンリーダーで理解できるか

## 主要コンポーネントカテゴリ

### Actions（アクション）

#### Buttons
- **用途**: ユーザーが明確なアクションを実行
- **種類**: Filled（最重要）、Outlined（中程度）、Text（低優先度）
- **いつ使う**: すべての主要アクション

#### FAB（Floating Action Button）
- **用途**: 画面の主要アクション
- **いつ使う**: 各画面に1つまで、最も重要なアクションのみ
- **例**: メール作成、新規投稿

#### Icon Buttons
- **用途**: コンパクトなアクション
- **いつ使う**: ツールバー、カード内のアクション
- **例**: いいね、共有、削除

### Navigation（ナビゲーション）

#### Navigation Bar
- **用途**: モバイルでの主要ナビゲーション
- **デバイス**: スマートフォン
- **推奨数**: 3-5個の目的地

#### Navigation Rail
- **用途**: タブレットでの主要ナビゲーション
- **デバイス**: タブレット、中型デバイス
- **推奨数**: 3-7個の目的地

#### Navigation Drawer
- **用途**: 大型デバイスでの包括的ナビゲーション
- **デバイス**: デスクトップ、大型タブレット
- **推奨数**: 5個以上の目的地も可能

#### Tabs
- **用途**: 同じ階層のコンテンツ切り替え
- **いつ使う**: 関連するコンテンツグループ
- **推奨数**: 2-6個のタブ

### Containment（コンテナ）

#### Cards
- **用途**: 関連情報のグループ化
- **種類**: Elevated（浮いている）、Filled、Outlined
- **いつ使う**: 複数の情報を1つのユニットとして表示

#### Lists
- **用途**: 同種の連続したアイテム
- **いつ使う**: 単純な繰り返しコンテンツ
- **例**: 連絡先リスト、設定項目

#### Divider
- **用途**: コンテンツの区切り
- **いつ使う**: リストやグループを視覚的に分離

### Communication（コミュニケーション）

#### Dialogs
- **用途**: 重要な情報や決定を要求
- **いつ使う**: ユーザーの即座の注意が必要
- **例**: 削除の確認、重要な通知

#### Snackbars
- **用途**: 軽量なフィードバック
- **いつ使う**: アクションの結果を通知（非破壊的）
- **例**: 「保存しました」「メッセージを送信しました」

#### Progress Indicators
- **用途**: ロード状態の表示
- **種類**: Linear（進行状況がわかる）、Circular（時間不明）

### Selection（選択）

#### Checkboxes
- **用途**: 複数選択
- **いつ使う**: オプションの選択・解除

#### Radio Buttons
- **用途**: 単一選択（相互排他）
- **いつ使う**: いくつかの選択肢から1つだけ選ぶ

#### Switches
- **用途**: オン/オフの切り替え
- **いつ使う**: 即座に効果が現れる設定

#### Chips
- **用途**: コンパクトな選択・フィルタ
- **種類**: Input（入力）、Filter（フィルタ）、Suggestion（提案）、Assist（アシスト）

### Text Input（テキスト入力）

#### Text Fields
- **用途**: テキスト入力
- **種類**: Filled、Outlined
- **いつ使う**: ユーザーからのテキスト入力が必要

## コンポーネント選択の決定ツリー

### ユーザーがアクションを実行したい

```
アクションの重要度は？
  高 → Filled Button / FAB
  中 → Outlined Button
  低 → Text Button / Icon Button
```

### ユーザーがナビゲートしたい

```
デバイスサイズは？
  モバイル（<600px） → Navigation Bar
  タブレット（600-1200px） → Navigation Rail
  デスクトップ（>1200px） → Navigation Drawer

同じページ内の切り替え？ → Tabs
```

### 情報を表示したい

```
コンテンツは？
  異なる種類の情報 → Card
  同種の繰り返し → List
  画像中心 → Card（画像付き）
  テキスト中心 → List
```

### ユーザーに選択させたい

```
選択の数は？
  単一選択 → Radio Buttons / Dropdown Menu
  複数選択 → Checkboxes / Chips
  オン/オフ → Switch
```

## ベストプラクティス

### DO（推奨）

1. **目的に応じた選択**: コンポーネントは意図に合わせて選ぶ
2. **一貫性**: 同じ目的には同じコンポーネント
3. **アクセシビリティ**: すべてのユーザーが使える
4. **シンプルさ**: 複雑な場合はシンプルなコンポーネントに分割

### DON'T（避けるべき）

1. **過剰なカスタマイズ**: ユーザーが認識できなくなる
2. **不適切なコンポーネント**: 例：アクションにCardを使用
3. **一貫性の欠如**: 同じ目的に異なるコンポーネント
4. **過度な装飾**: 機能を損なう視覚的装飾
