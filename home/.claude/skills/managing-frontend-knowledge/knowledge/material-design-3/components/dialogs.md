# Dialogs（ダイアログ）

ダイアログは、ユーザーに重要な情報を提示したり、決定を求めたりする一時的なUIです。

## いつ使うべきか

### Dialogを使うべき場合

- **即座の注意が必要**: ユーザーのワークフローを中断して対応が必要
- **重要な決定**: 削除の確認など、取り返しのつかない操作の確認
- **必須の情報入力**: 続行するために必要な情報の入力
- **重要な通知**: エラーや警告など、見逃せない情報

### Dialogを避けるべき場合

- **軽微な通知**: Snackbarで十分
- **長い情報**: 新しいページやボトムシートが適切
- **頻繁な使用**: ユーザーの邪魔になる
- **複雑なフォーム**: 専用ページで処理すべき

## ダイアログの種類

### 1. Basic Dialog（基本ダイアログ）
- **用途**: シンプルな選択や確認
- **例**: 「削除しますか？」

### 2. Alert Dialog（アラートダイアログ）
- **用途**: 重要な警告や情報
- **例**: エラーメッセージ、重要な通知

### 3. Confirmation Dialog（確認ダイアログ）
- **用途**: アクションの確認
- **例**: 「本当に削除しますか？」

### 4. Full-screen Dialog（全画面ダイアログ）
- **用途**: 複雑な入力や編集
- **例**: フォームの編集、複数ステップのプロセス

## ダイアログの構造

### 基本要素

1. **Container**: ダイアログの背景（最高のElevation）
2. **Title**: ダイアログの目的を明確に
3. **Content**: 説明や入力フィールド
4. **Actions**: 確認、キャンセルボタン

### アクションボタンの配置

```
[Cancel（Text/Outlined）]  [Confirm（Filled）]
```

- 確認アクションは右側、Filledボタン
- キャンセルは左側、TextまたはOutlinedボタン
- 破壊的アクション（削除など）はError色

## ベストプラクティス

### DO（推奨）

1. **明確なタイトル**: 何についてのダイアログか一目でわかる
2. **簡潔な内容**: 1-2文で要点を伝える
3. **明確なアクション**: 「削除」「キャンセル」など具体的に
4. **Escapeキーでクローズ**: キーボードでも閉じられる
5. **背景をクリックでクローズ**: 直感的な操作

### DON'T（避けるべき）

1. **曖昧なタイトル**: 「確認」だけでは不十分
2. **長すぎる内容**: スクロールが必要なダイアログは避ける
3. **曖昧なボタン**: 「OK」「はい」ではなく具体的に
4. **頻繁な使用**: ユーザーの邪魔にならないよう最小限に
5. **複雑なフォーム**: ダイアログではなく専用ページで

## アクセシビリティ

- **フォーカストラップ**: ダイアログ内でTa bキーが循環
- **初期フォーカス**: 最初のフォーカス可能な要素にフォーカス
- **Escapeキー**: Escapeでダイアログをクローズ
- **スクリーンリーダー**: ダイアログのタイトルと役割を読み上げ
