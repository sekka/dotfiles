# Cards（カード）

カードは、関連する情報をグループ化して表示するコンテナコンポーネントです。

## カードの種類

### 1. Elevated Card（エレベートカード）
- **特徴**: 影で浮いている
- **用途**: 標準的なカード表示
- **背景**: ライトモードの白背景に適している

### 2. Filled Card（塗りつぶしカード）
- **特徴**: 背景色で区別
- **用途**: カラフルな背景や、複数のカードを区別
- **背景**: 任意の背景色に適応

### 3. Outlined Card（アウトラインカード）
- **特徴**: 枠線で区別
- **用途**: 控えめな区切り
- **背景**: ダークモードやフラットデザインに適している

## いつ使うべきか

### Cardを使うべき場合

- **異なる種類の情報**: 画像、タイトル、説明、アクションなどを含む
- **クリック可能なコンテンツ**: カード全体がクリックできる
- **独立したユニット**: 各カードが独立した情報のまとまり

### Listを使うべき場合

- **同種の情報の繰り返し**: 連絡先リスト、設定項目など
- **シンプルな構造**: テキストとアイコンのみ
- **スキャン性重視**: ユーザーが素早く目的の項目を見つける

## カードの構造

### 基本要素

1. **Container**: カードの背景とエレベーション
2. **Media（オプション）**: 画像や動画
3. **Header（オプション）**: タイトル、サブタイトル
4. **Supporting text（オプション）**: 説明文
5. **Actions（オプション）**: ボタンやアイコンボタン

### レイアウトパターン

#### パターン1: メディア中心
```
[画像（大）]
[タイトル]
[説明]
[アクション]
```

#### パターン2: テキスト中心
```
[タイトル]
[サブタイトル]
[説明文（長め）]
[アクション]
```

#### パターン3: リスト風
```
[アイコン] [タイトル] [サブタイトル]
[アクション]
```

## ベストプラクティス

### DO（推奨）

1. **明確な階層**: タイトルを強調、説明は控えめに
2. **アクションは最小限**: 1-2個のアクションボタン
3. **一貫性**: 同じコンテキストでは同じカードタイプ
4. **適切なスペーシング**: カード間に十分な余白

### DON'T（避けるべき）

1. **過剰な情報**: カードに詰め込みすぎない
2. **複数の主要アクション**: 3つ以上のボタンは避ける
3. **不適切な使用**: リストで十分な場合はカードを使わない
