# Buttons（ボタン）

ボタンは、ユーザーがアクションを実行するための最も基本的なコンポーネントです。

## ボタンの種類

### 1. Filled Button（塗りつぶしボタン）
- **用途**: 最も重要なアクション
- **視覚的重み**: 最も高い
- **使用頻度**: 1画面に1つが理想
- **例**: 「送信」「保存」「購入」

### 2. Filled Tonal Button（トーナル塗りつぶしボタン）
- **用途**: Filledより若干控えめなアクション
- **視覚的重み**: 高い
- **使用頻度**: 中程度
- **例**: 「次へ」「詳細を見る」

### 3. Outlined Button（アウトラインボタン）
- **用途**: 中程度の重要性のアクション
- **視覚的重み**: 中
- **使用頻度**: 中〜高
- **例**: 「キャンセル」「戻る」

### 4. Text Button（テキストボタン）
- **用途**: 最も控えめなアクション
- **視覚的重み**: 低い
- **使用頻度**: 高
- **例**: 「スキップ」「もっと見る」「学ぶ」

### 5. Elevated Button（エレベートボタン）
- **用途**: 視覚的に浮いた状態のボタン
- **使用場面**: カラフルな背景に配置する場合
- **視覚的重み**: 中〜高

## いつどのボタンを使うべきか

### 決定ツリー

```
アクションの重要度は？
  ↓
最も重要（画面の主要アクション）
  → Filled Button
  例: フォーム送信、決済確定

重要（でも主要ではない）
  → Filled Tonal Button or Outlined Button
  例: 「次へ」「編集」

補助的・キャンセル
  → Outlined Button or Text Button
  例: 「キャンセル」「戻る」

最小限の強調
  → Text Button
  例: 「スキップ」「詳細」

背景がカラフル
  → Elevated Button
  例: 画像の上に配置
```

### ダイアログでのボタン配置

```
[Cancel（Text）]  [Confirm（Filled）]
```
- 確認アクションは右側でFilled
- キャンセルは左側でText or Outlined

### フォームでのボタン配置

```
[Reset（Outlined）]  [Submit（Filled）]
```
- 主要アクションは右側
- 破壊的アクションは左側で控えめに

## ボタンのラベル

### DO（推奨）

- **動詞で始める**: 「保存」「削除」「送信」
- **短く明確に**: 1-2単語が理想
- **具体的に**: 「OK」より「保存」
- **結果を示す**: 「次へ進む」「変更を保存」

### DON'T（避けるべき）

- **曖昧な表現**: 「OK」「はい」（何に対して？）
- **長すぎる**: 「このフォームを送信して次のステップに進む」
- **技術用語**: 「Submit」より「送信」（日本語UIの場合）

## アクセシビリティ

### 最小タップ領域
- **モバイル**: 48x48dp（44x44pt iOS）
- **デスクトップ**: マウスなので柔軟だが、40x40px以上推奨

### コントラスト
- ボタンラベルと背景: 4.5:1以上
- ボタンと周囲: 3:1以上（アウトラインボタン）

### フォーカス状態
- キーボードフォーカスで明確な視覚的インディケータ
- Tabキーでアクセス可能

### スクリーンリーダー
- ボタンの目的が明確
- アイコンのみの場合は aria-label 必須

## ベストプラクティス

### DO（推奨）

1. **優先順位を明確に**: 最も重要なボタンを視覚的に強調
2. **一貫性**: 同じアクションには同じスタイル
3. **適切な配置**: 主要アクションは右下または右側
4. **フィードバック**: クリック時の視覚的応答（Ripple effect）

### DON'T（避けるべき）

1. **複数のFilledボタン**: 1画面に2つ以上のFilledは混乱を招く
2. **長すぎるラベル**: ボタンの幅が広くなりすぎる
3. **アイコンの乱用**: すべてのボタンにアイコンは不要
4. **不適切なカラー**: 破壊的アクション（削除）に緑色など
