# Section I

Terminology, usage conventions, and writing guidelines for Apple product documentation beginning with "I".

## Key Terms and Definitions

**I-beam**: A pointer reference used in technical documentation.

**iBooks**: Use for versions before iOS 12 and macOS Mojave; switch to "Apple Books" for later versions.

**iCloud**:
- Capitalize the "C" in all references
- Use "in" for stored content; "on" for iCloud.com access
- Users "sign in to" their Apple Account, not "sign in to iCloud"
- iCloud "syncs" content between devices
- Users "upgrade" storage, not "purchase" it

**iCloud Drive**: Feature allowing document storage accessible across multiple devices.

**iCloud Photos**: The current term; avoid "iCloud Photo Library."

**iCloud+**: Subscription service; don't use "iCloud Plus."

**Icon**:
- Unnamed icons should be described by appearance or function in lowercase
- Icons that function as buttons are called "buttons"
- App icons remain "icons" even when clickable

**iPhone/iPad**:
- Don't use articles in general references
- Use device names instead of "mobile device"
- List devices consistently: "iPhone, iPad, and Mac"
- Don't shorten iPhone to "phone"

**iOS/iPadOS**: Operating systems used without articles; distinguish app versions using "[app name] for [platform]" format.

**IT Terms**:
- IMAP: "Internet Message Access Protocol"
- ISP: "internet service provider"
- IP: "Internet Protocol"
- I/O: "input/output"

## Writing Guidelines

- Replace "i.e." with "that is"
- Use "if" for conditions; "whether" when outcomes are possible
- Use "into" for motion; "in to" when "in" is part of the verb
- Avoid "impairment"; use specific disability terms instead
- Use "information" instead of "data" in user materials
- Replace "IM" with "message"
- Use "enter" or "type" instead of "input" as a verb

## Notable Conventions

**Important Notices**: Reserve for significant trouble spots not involving injury or data loss.

**Italics Usage**:
- Document titles
- Letters and words as examples
- Placeholder names
- Variables in code

**Inclusive Language**: Emphasized throughout with dedicated sections on gender identity and disability representation.

This guide prioritizes clarity, consistency, and accessibility in Apple's technical communications.
