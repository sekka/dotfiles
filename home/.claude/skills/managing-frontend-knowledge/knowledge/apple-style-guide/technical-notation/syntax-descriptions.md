# Syntax Descriptions

Standards for describing technical syntax in Apple materials.

## Purpose

Syntax descriptions help developers understand:
- Required vs. optional parameters
- Parameter types and formats
- Valid syntax patterns
- Command structure

## Conventions

### Notation Elements

- **Bold or monospace**: Required elements
- **Italic or brackets**: Optional elements
- **Vertical bar (|)**: Alternative options
- **Ellipsis (...)**: Repeatable elements

### Parameter Descriptions

- Clearly indicate data types
- Specify valid values or ranges
- Note default values when applicable
- Explain parameter purposes

## Examples

Provide clear, practical examples that demonstrate:
- Common use cases
- Edge cases when relevant
- Best practices
- Expected output
