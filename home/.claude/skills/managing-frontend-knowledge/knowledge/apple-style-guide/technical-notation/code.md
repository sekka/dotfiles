# Code

Guidelines for displaying code examples and snippets in Apple materials.

## General Principles

- Use monospace font for all code examples
- Maintain proper indentation to show code structure
- Include syntax highlighting when appropriate
- Keep code examples concise and focused

## Code Block Formatting

- Use code blocks for multi-line code examples
- Include language identifier for syntax highlighting
- Ensure code is syntactically correct and runnable when possible

## Best Practices

- Comment code when necessary for clarity
- Use realistic examples that developers can relate to
- Follow language-specific conventions and style guides
- Test code examples before publication

## Inline Code

- Use code font for short code snippets within sentences
- Keep inline code brief and readable
- Don't break inline code across lines
