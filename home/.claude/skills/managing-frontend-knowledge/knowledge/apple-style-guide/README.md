# Apple Style Guide

Apple Style Guideの日本語版ナレッジベース。Appleの文書、参照資料、トレーニング、ユーザーインターフェースにおいて一貫性のある表現を維持するための編集ガイドライン。

## 概要

Apple Style Guideは、Apple製品のドキュメント作成において統一されたトーン＆マナーを維持するための公式ガイドラインです。ライター、編集者、開発者、サードパーティ開発者を対象としています。

**公式サイト**: https://support.apple.com/ja-jp/guide/applestyleguide/welcome/web
**最終更新**: June 2025

## 推奨参考資料

- **Merriam-Webster's Collegiate Dictionary**: スペルの基準
- **The Chicago Manual of Style**: スタイルと用法の基準

## 使い方

### 特定のトピックを検索する場合

1. **製品名の使い方**: `style-usage/a.md`、`style-usage/i.md`、`style-usage/m.md`を参照
2. **インクルーシブな表現**: `writing-inclusively/`ディレクトリ全体を確認
3. **コードの書き方**: `technical-notation/`セクションを参照
4. **国際化対応**: `international-style/`セクションを確認

### 主要なガイドライン

#### 文体の原則
- 能動態を使用する
- 一般的な短縮形は使用可（aren't、isn't、there's等）
- ユーザーに直接語りかける（"you can..."）

#### 製品名
- Apple製品名は省略しない
- 正確な大文字小文字を使用（iPhone、iPad、Mac、macOS）
- 商標記号を適切に使用

#### インクルーシビティ
- ジェンダーニュートラルな言葉を優先
- 単数形のthey/their/themを使用可
- 障害については具体的な用語を使用
- 多様性を反映する例を使用

#### 技術表記
- コード要素にはモノスペースフォントを使用
- プレースホルダーにはイタリック体を使用
- 構文説明には標準的な表記法を使用

## 重要な注意事項

- このガイドはU.S. English規則に基づいています
- ローカライズコンテンツには地域固有のリソースが必要です
- Appleの部門によっては補足ガイドがある場合があります
- ハイフネーションルールに特に注意（名詞の前ではハイフン、述語位置では通常ハイフンなし）
