# 出典管理パターン

## 概要

クライアントヒアリング整理において、情報の出典を明確にすることは信頼性の基盤となる。
このドキュメントでは、情報源ごとの表記ルールと確度の評価基準を定義する。

---

## 基本原則

### 1. 透明性の確保

- 全ての情報は出典を明記する
- 推測・仮定は明確に区別する
- 確度の低い情報は「要確認」とマークする

### 2. 検証可能性

- 外部調査には必ずURLを付記
- クライアント資料にはページ番号を付記
- ヒアリングには発言者・日時を記録（可能な範囲で）

### 3. 階層的な表記

- 情報の重要度に応じて表記方法を変える
- 読みやすさを優先（過度な注釈は避ける）

---

## 出典表記ルール

### パターン1: ヒアリング由来

**表記**: `※ヒアリングより`

**確度**: 要確認（口頭情報）

**使用例**:

```markdown
> 現在の主要顧客は30-40代の女性層が中心。
> ※ヒアリングより（2026-01-23 担当者A氏）
```

**注意事項**:
- 口頭情報は誤解や記憶違いの可能性がある
- 重要な数値・日付は後日書面で確認
- 発言者名は守秘義務に配慮（必要に応じて匿名化）

---

### パターン2: クライアント資料

**表記**: `※資料p.Xより`

**確度**: 高（公式資料）

**使用例**:

```markdown
2025年度の売上目標は前年比120%（5億円）。
※資料p.12より（会社説明資料 v2.3）
```

**注意事項**:
- ページ番号は必須
- 資料名・バージョンも記載（可能な範囲で）
- 図表からの読み取りは「資料p.X 図表より」と明記

---

### パターン3: 外部調査（Web）

**表記**: 脚注形式 `[^XX]` + 文末にURL

**確度**: 中（公開情報）

**使用例**:

```markdown
国内ブランディング市場は2025年に約3,000億円規模と推定されている[^1]。

---

[^1]: 日経クロストレンド「ブランディング市場調査2025」https://example.com/article
```

**注意事項**:
- 信頼性の高い情報源を優先（公的機関、業界団体、大手メディア）
- 個人ブログは避ける（専門家の意見として引用する場合のみ）
- アクセス日時を記録（リンク切れ対策）

---

### パターン4: 推定・仮定

**表記**: `※推定（根拠：〜）`

**確度**: 低（要検証）

**使用例**:

```markdown
月間アクティブユーザー数は約10,000人と推定される。
※推定（根拠：GA直帰率60%、セッション数から逆算）
```

**注意事項**:
- 推定の根拠を必ず明記
- 検証方法を併記（どうすれば確認できるか）
- 重要な意思決定には使わない

---

### パターン5: 一般知識・補足説明

**表記**: `📝補足` または注釈なし

**確度**: 高（常識的知識）

**使用例**:

```markdown
SEO対策では、Googleの検索アルゴリズムに最適化することが重要。
📝補足: SEOはSearch Engine Optimizationの略。検索結果の上位表示を目指す施策。
```

**注意事項**:
- 専門用語の解説に使用
- 業界常識の補足に使用
- 過度な説明は避ける（読みやすさ優先）

---

## 確度評価基準

| レベル | 定義 | 対応する出典 | 検証方法 |
|--------|------|--------------|----------|
| **高** | 公式文書または一般的事実 | クライアント資料、一般知識 | 不要（既に検証済み） |
| **中** | 信頼性のある第三者情報 | 外部調査（Web） | 情報源の信頼性確認 |
| **低（要確認）** | 口頭情報または推定 | ヒアリング、推定 | 書面確認、実測 |

---

## 出典の優先順位

情報が複数の出典で矛盾する場合の判断基準：

1. **クライアント資料**（公式見解）
2. **ヒアリング**（最新情報の可能性）
3. **外部調査**（参考情報）
4. **推定**（最も不確実）

**対応フロー**:

```
矛盾を発見
  ↓
クライアント資料を優先
  ↓
矛盾点を明記「※資料では〇〇だが、ヒアリングでは××と発言」
  ↓
AskUserQuestionでユーザーに確認依頼
  ↓
確認後、正しい情報を本文に、誤った情報は削除or注釈へ移動
```

## セクション別の出典管理

### 数値データ

- **必須**: 出典 + 測定方法 + 時期

**例**:
```markdown
月間PV: 50,000
※資料p.5より（Google Analytics 2025年12月実績）
```

### 定性的情報

- **必須**: 出典 + 発言者（可能な範囲で）

**例**:
```markdown
> ブランドイメージは「信頼できる」「堅実」といった言葉で表現される。
> ※ヒアリングより（マーケティング責任者）
```

### 競合情報

- **必須**: 情報源 + 取得日

**例**:
```markdown
競合A社は2025年にリブランディングを実施[^2]。

[^2]: 日経ビジネス「A社の戦略転換」(2025-11-10) https://example.com
```

## 実装時の注意点

### 1. 可読性とのバランス

- 過度な出典表記は読みにくくなる
- 重要な情報にのみ出典を付記
- 一般的な事実には注釈不要

**NG例**:
```markdown
Webサイト[^1]は企業[^2]にとって重要[^3]なマーケティングツール[^4]です。
```

**OK例**:
```markdown
Webサイトは企業にとって重要なマーケティングツールです。
```

### 2. 出典の集約

- 同じ資料から複数の情報を引用する場合、セクション末尾にまとめて記載

**例**:
```markdown
## ターゲット分析

- 年齢層: 30-40代
- 性別: 女性70%、男性30%
- 居住地: 首都圏60%、地方40%

※上記データは資料p.8-10より（顧客分析レポート 2025年版）
```

### 3. 更新履歴の管理

- 情報が更新された場合、旧情報は削除せず履歴として残す

**例**:
```markdown
## 事業規模

年商: 5億円（2025年実績）
※資料p.3より

---

**更新履歴**:
- 2026-01-20: 初回ヒアリングでは「約4億円」と聞いていたが、資料で5億円と確認
```

## チェックリスト

文書作成時に以下を確認：

- [ ] 数値データに出典が付記されている
- [ ] ヒアリング情報に「要確認」マークがついている
- [ ] 外部調査にURLが付記されている
- [ ] 推定・仮定に根拠が明記されている
- [ ] 矛盾する情報が整理されている
- [ ] 出典が多すぎて読みにくくなっていない

---

## まとめ

出典管理は、クライアントとの信頼関係を構築し、後の検証作業をスムーズにするための基盤。
透明性と可読性のバランスを取りながら、適切に情報源を明記すること。
