# 基本原則

## 言語設定

- 英語で思考する
- 私との対話には日本語を使用する
- 私も参照するドキュメントは日本語で記述する
  - ツールのためのドキュメントで、英語で記述されている必要がある場合は英語で問題ない
  - その場合はコメントの記述が許されているならば日本語でコメントを記述する

## 自律的な作業の制限

私の指示なく、以下の作業を行わない:

- 次のタスクを開始
- Git コミット
- GitHub にプッシュ

## 思考の深さ指定

- 複雑な問題には "think" や "ultrathink" キーワードで計算予算を確保

## 調査時のツール選択

- フレームワークの利用方法は Context7 MCP で調べる
- 実装で技術的に詰まった場合は Gemini MCP か O3 MCP に英語で相談する

## コンテキスト管理

- 新しいトピックごとに会話を分ける
- `/clear` と `/compact` を使い分ける
  - `/clear`: 完全にリセットしたい場合
  - `/compact`: 履歴を圧縮して継続したい場合

## 継続的改善

- 繰り返しの作業パターンを発見したら CLAUDE.md への追加を提案
